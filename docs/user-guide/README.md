## 快速开始

### 登录系统

1. 访问 SecFlow 管理界面
2. 输入用户名和密码
3. 点击"登录"按钮

> **首次使用**: 默认管理员账号为 `admin`，密码在部署时设置。

### 首页仪表盘

登录后进入首页仪表盘，显示：

- **统计卡片**：今日新增漏洞、本周漏洞趋势
- **热点漏洞**：最新高危漏洞列表
- **节点状态**：当前在线节点数
- **任务状态**：运行中/已完成任务统计

## 漏洞情报

### 浏览漏洞

1. 点击左侧菜单"漏洞情报"
2. 查看漏洞列表，支持分页浏览
3. 使用筛选器缩小范围

**筛选选项**：

- **风险等级**：严重、高危、中危、低危
- **数据来源**：avd-rod、seebug-rod、kev-rod 等
- **关键词**：搜索漏洞标题、CVE 编号
- **是否已推送**：筛选推送状态

### 查看漏洞详情

1. 点击漏洞标题进入详情页
2. 查看完整的漏洞信息：
   - 漏洞标题和描述
   - CVE 编号
   - 风险等级
   - 披露时间
   - 解决方案
   - 参考链接
   - GitHub 搜索关键词

### 导出漏洞

1. 在漏洞列表页面点击"导出"按钮
2. 选择筛选条件
3. 点击"导出 CSV"
4. 下载生成的 CSV 文件

### 删除漏洞

1. 在漏洞列表中点击删除图标
2. 确认删除操作
3. 漏洞将被永久删除

> **注意**：删除操作需要 Editor 或以上权限。

## 节点管理

### 查看节点状态

1. 点击"系统管理" → "节点管理"
2. 查看所有注册的客户端节点

**节点状态说明**：

| 状态 | 说明 |
|------|------|
| 在线 | 节点正常运行，可接收任务 |
| 离线 | 节点未连接或已断开 |
| 忙碌 | 节点正在执行任务 |
| 暂停 | 节点被管理员暂停 |

### 节点详情

每个节点显示：

- **基本信息**：名称、ID、注册时间
- **硬件信息**：IP、操作系统、CPU、内存、磁盘
- **性能指标**：CPU 使用率、内存使用率
- **任务统计**：总任务数、成功数、失败数、平均响应时间
- **负责数据源**：该节点负责的爬虫源

### 节点操作

| 操作 | 说明 |
|------|------|
| 暂停 | 暂停节点接收新任务 |
| 恢复 | 恢复节点接收任务 |
| 断开 | 强制断开节点连接 |
| 刷新 Token | 重新生成节点认证 Token |
| 删除 | 从系统中移除节点 |

> **注意**：节点操作需要 Admin 权限。

### 添加新节点

1. 点击"创建节点"按钮
2. 输入节点名称
3. 选择该节点负责的数据源
4. 点击"创建"
5. 复制生成的 Token 并配置到客户端

## 任务管理

### 查看任务列表

1. 点击"系统管理" → "任务管理"
2. 查看所有任务记录

**任务状态说明**：

| 状态 | 说明 |
|------|------|
| pending | 等待调度 |
| dispatched | 已分发到节点 |
| running | 节点正在执行 |
| done | 任务完成 |
| failed | 任务失败 |

### 创建爬取任务

#### 创建漏洞爬取任务

1. 点击"创建任务" → "漏洞爬取"
2. 选择数据源（可多选）
3. 设置页面限制（每个源爬取页数）
4. 设置优先级
5. 设置超时时间
6. 点击"创建"

#### 创建文章爬取任务

1. 点击"创建任务" → "文章爬取"
2. 选择数据源
3. 设置页面限制
4. 点击"创建"

### 任务详情

每个任务显示：

- **基本信息**：任务 ID、类型、创建时间
- **执行状态**：当前状态、进度
- **分配信息**：分配的节点
- **执行结果**：成功/失败、爬取数量
- **重试信息**：重试次数、错误历史

### 任务操作

| 操作 | 说明 |
|------|------|
| 停止 | 停止正在运行的任务 |
| 删除 | 删除任务记录 |
| 重试 | 重新执行失败的任务 |

### 死信队列

无法成功执行的任务会进入死信队列：

1. 点击"系统管理" → "死信队列"
2. 查看永久失败的任务
3. 可选择重试或删除

## 推送配置

### 添加推送渠道

1. 点击"系统管理" → "推送渠道"
2. 点击"添加渠道"

#### 钉钉

1. 选择类型"钉钉"
2. 输入名称（如"告警群"）
3. 输入 Webhook 地址
4. 输入 Secret（可选，用于加签）
5. 点击"保存"

#### 飞书

1. 选择类型"飞书"
2. 输入名称
3. 输入 Webhook 地址
4. 点击"保存"

#### 企业微信

1. 选择类型"企业微信"
2. 输入名称
3. 输入 Webhook 地址
4. 点击"保存"

#### Telegram

1. 选择类型"Telegram"
2. 输入名称
3. 输入 Bot Token
4. 输入 Chat ID
5. 点击"保存"

#### Slack

1. 选择类型"Slack"
2. 输入名称
3. 输入 Webhook 地址
4. 点击"保存"

#### 自定义 Webhook

1. 选择类型"Webhook"
2. 输入名称
3. 输入目标 URL
4. 选择请求方法（POST/GET）
5. 添加自定义请求头（如 Authorization）
6. 点击"保存"

### 测试推送

创建渠道后，点击"测试"按钮发送测试消息，验证配置是否正确。

### 管理推送渠道

- **启用/禁用**：点击开关切换
- **编辑**：修改渠道配置
- **删除**：移除推送渠道

## 常见问题

### Q: 无法登录

**可能原因**：

1. 用户名或密码错误
2. 账户已被禁用
3. 服务器未启动

**解决方案**：

- 确认用户名密码正确
- 联系管理员检查账户状态
- 检查服务端日志

### Q: 节点显示离线

**可能原因**：

1. 客户端程序未运行
2. 网络连接问题
3. Token 配置错误
4. 服务端地址配置错误

**解决方案**：

1. 检查客户端服务状态
2. 测试网络连通性：`ping server-ip`
3. 核对客户端 Token 与服务端一致
4. 检查客户端日志

### Q: 任务一直处于 pending

**可能原因**：

1. 没有在线的客户端节点
2. 节点未配置相关数据源
3. 调度器未启动

**解决方案**：

1. 确保至少有一个节点在线
2. 检查节点配置的数据源
3. 重启服务端调度器

### Q: 推送消息发送失败

**可能原因**：

1. Webhook 地址配置错误
2. 机器人被禁言/踢出群
3. 网络问题

**解决方案**：

1. 核实 Webhook 地址
2. 在群中测试机器人权限
3. 使用"测试"功能排查问题

### Q: 如何备份数据

**MongoDB 备份**：

```bash
mongodump "mongodb://user:pass@host:27017/secflow" \
  --out /backup/path
```

**Redis 备份**：

```bash
redis-cli -a password SAVE
```

### Q: 如何升级系统

1. **停止服务**：`systemctl stop secflow-server`
2. **备份数据**
3. **更新代码**：`git pull`
4. **重新编译**
5. **启动服务**：`systemctl start secflow-server`
