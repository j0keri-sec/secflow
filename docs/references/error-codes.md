# 错误码参考

本文档列出 SecFlow 系统的所有错误码及其处理建议。

## 错误响应格式

```json
{
  "code": 1001,
  "message": "参数错误",
  "detail": "page 参数必须为正整数",
  "request_id": "req_abc123"
}
```

## 错误码分类

| 范围 | 分类 |
|------|------|
| 1xxx | 认证与授权错误 |
| 2xxx | 资源相关错误 |
| 3xxx | 任务相关错误 |
| 4xxx | 节点相关错误 |
| 5xxx | 业务操作错误 |
| 9xxx | 系统内部错误 |

---

## 认证与授权错误 (1xxx)

### 1001 - 参数错误

**原因**：请求参数格式不正确或缺少必需参数。

**解决**：
- 检查请求 JSON 格式
- 确保必需字段已填写
- 验证参数类型和范围

```json
// 错误示例
{"code": 1001, "message": "参数错误", "detail": "page 参数必须为正整数"}

// 正确示例
{"page": 1, "page_size": 20}
```

### 1002 - 认证失败

**原因**：Token 无效、过期或格式错误。

**解决**：
- 重新登录获取新 Token
- 检查 Token 格式是否正确（Bearer xxx）
- 确保 Token 未过期

```bash
# 刷新 Token
curl -X POST http://localhost:8080/api/v1/auth/refresh \
  -H "Authorization: Bearer <refresh_token>"
```

### 1003 - 权限不足

**原因**：当前用户/节点没有执行此操作的权限。

**解决**：
- 联系管理员提升权限
- 使用具有足够权限的账号
- 检查操作是否针对正确的资源

### 1004 - Token 过期

**原因**：访问 Token 已过期。

**解决**：
- 使用 Refresh Token 获取新 Token
- 重新登录

### 1005 - 无效 Refresh Token

**原因**：Refresh Token 无效或已撤销。

**解决**：
- 需要重新登录获取新凭证

## 资源相关错误 (2xxx)

### 2001 - 资源不存在

**原因**：请求的资源（漏洞/文章/任务/节点）在数据库中不存在。

**解决**：
- 检查 ID 是否正确
- 确认资源是否已被删除
- 使用列表接口确认资源存在

```bash
# 确认资源存在
curl http://localhost:8080/api/v1/vulns \
  -H "Authorization: Bearer $TOKEN" | jq '.data.items[] | select(.id=="vuln_xxx")'
```

### 2002 - 资源已存在

**原因**：尝试创建的资源已存在（唯一性冲突）。

**解决**：
- 检查是否重复创建
- 使用更新接口而非创建接口
- 修改标识符后重试

### 2003 - 资源被占用

**原因**：资源正被其他操作使用，无法删除或修改。

**解决**：
- 等待其他操作完成
- 强制释放资源（如停止运行中的任务）

## 任务相关错误 (3xxx)

### 3001 - 任务不存在

**原因**：指定的任务 ID 在队列中不存在。

**解决**：
- 检查任务 ID 是否正确
- 任务可能已完成或被删除
- 查看任务列表确认

```bash
curl http://localhost:8080/api/v1/tasks \
  -H "Authorization: Bearer $TOKEN" | jq '.data.items[].id'
```

### 3002 - 任务执行失败

**原因**：任务在执行过程中发生错误。

**解决**：
- 查看任务详情获取错误信息
- 检查节点日志
- 可能需要增加超时时间或重试

```bash
# 查看任务详情
curl http://localhost:8080/api/v1/tasks/:id \
  -H "Authorization: Bearer $TOKEN" | jq '.data.error'
```

### 3003 - 任务已取消

**原因**：任务被用户或系统取消。

**解决**：
- 需要重新创建任务
- 检查是否有触发取消的操作

### 3004 - 任务状态无效

**原因**：无法对处于当前状态的任务执行此操作。

**常见场景**：
- 尝试停止已完成的任務
- 尝试重启正在运行的任务
- 尝试删除运行中的任务

**解决**：
- 等待任务状态变更
- 先停止任务再执行其他操作

### 3005 - 任务超时

**原因**：任务执行时间超过设定的超时时间。

**解决**：
- 增加超时时间配置
- 减少 page_limit
- 检查目标站点响应速度
- 使用性能更好的节点

### 3006 - 任务队列已满

**原因**：待处理任务数量达到上限。

**解决**：
- 等待队列处理完成
- 增加调度频率
- 扩容节点数量

## 节点相关错误 (4xxx)

### 4001 - 节点不在线

**原因**：目标节点未建立 WebSocket 连接。

**解决**：
- 检查节点是否运行
- 检查网络连通性
- 查看节点日志
- 重启节点客户端

```bash
# 检查节点状态
curl http://localhost:8080/api/v1/nodes \
  -H "Authorization: Bearer $TOKEN" | jq '.data.items[].status'
```

### 4002 - 节点繁忙

**原因**：节点并发任务数已达上限。

**解决**：
- 等待节点空闲
- 将任务分配给其他节点
- 增加节点的 max_concurrent 配置

### 4003 - 节点未注册

**原因**：节点未在服务端注册。

**解决**：
- 确认节点配置正确
- 重启节点客户端
- 检查服务端是否正常运行

### 4004 - 节点心跳超时

**原因**：节点长时间未发送心跳。

**解决**：
- 检查节点网络状况
- 增加心跳间隔配置
- 查看节点日志排查原因

### 4005 - 节点认证失败

**原因**：节点 Token 验证失败。

**解决**：
- 检查节点 Token 配置
- 重新生成节点 Token
- 检查服务端 JWT 配置

## 业务操作错误 (5xxx)

### 5001 - 推送失败

**原因**：向推送渠道发送消息失败。

**细分错误**：
- `5001.1` - Webhook URL 不可达
- `5001.2` - 推送内容格式错误
- `5001.3` - 推送渠道配置无效
- `5001.4` - 推送频率超限

**解决**：
```bash
# 测试推送渠道
curl -X POST http://localhost:8080/api/v1/channels/:id/test \
  -H "Authorization: Bearer $TOKEN"
```

### 5002 - 爬取失败

**原因**：爬取目标站点失败。

**细分错误**：
- `5002.1` - 网络连接失败
- `5002.2` - 页面解析失败
- `5002.3` - 数据提取失败

**解决**：
- 检查网络连通性
- 查看节点日志获取详细错误
- 可能需要更新爬取器代码

### 5003 - 目标站点不可达

**原因**：无法访问目标网站。

**解决**：
- 检查网络代理配置
- 确认目标站点是否正常
- 尝试更换节点网络环境

### 5004 - WAF 拦截

**原因**：目标站点的 WAF 拒绝访问。

**解决**：
- 启用代理池
- 增加请求间隔
- 调整 User-Agent
- 启用 Stealth 模式

```yaml
# 客户端配置
browser:
  stealth: true
  proxy: "http://proxy-pool:8080"
crawler:
  delay_min: 2000
  delay_max: 5000
```

### 5005 - 验证码拦截

**原因**：目标站点要求输入验证码。

**解决**：
- 使用人机验证解决方案
- 增加请求间隔
- 更换 IP 代理
- 标记该站点需要特殊处理

### 5006 - 数据格式错误

**原因**：目标站点返回的数据格式与预期不符。

**解决**：
- 更新爬取器的解析逻辑
- 查看节点日志获取实际返回
- 可能目标站点已更新页面结构

## 系统内部错误 (9xxx)

### 9001 - 数据库错误

**原因**：MongoDB 操作失败。

**解决**：
- 检查 MongoDB 连接
- 查看 MongoDB 日志
- 可能是数据库连接池耗尽

### 9002 - 缓存错误

**原因**：Redis 操作失败。

**解决**：
- 检查 Redis 连接
- 查看 Redis 日志
- 确认 Redis 内存充足

### 9003 - 内部服务错误

**原因**：服务端内部逻辑错误。

**解决**：
- 查看服务端日志
- 可能需要重启服务
- 联系技术支持

### 9004 - 依赖服务不可用

**原因**：依赖的外部服务（如消息队列）不可用。

**解决**：
- 检查相关服务状态
- 查看详细错误日志
- 确认网络连通性

## 常见错误场景与处理

### 场景 1：新建任务后立即显示失败

```
排查步骤：
1. 检查节点是否在线
2. 查看任务详情错误信息
3. 检查节点日志
4. 确认网络连通性
```

### 场景 2：推送消息收不到

```
排查步骤：
1. 测试推送渠道是否正常
2. 检查 Webhook URL 是否正确
3. 确认渠道是否启用
4. 查看服务端推送日志
```

### 场景 3：节点长时间不接收任务

```
排查步骤：
1. 检查节点状态是否 online
2. 查看节点并发限制
3. 检查节点能力标签
4. 确认任务与节点能力匹配
```

### 场景 4：爬取数据不完整

```
排查步骤：
1. 检查是否被 WAF 拦截
2. 查看节点日志的错误信息
3. 确认页面结构是否变化
4. 验证目标站点的反爬策略
```

## 日志定位

### 服务端日志关键字

```bash
# 查看错误日志
grep -E "ERROR|error" server.log | tail -100

# 查看特定错误码
grep "5002" server.log

# 查看节点相关错误
grep -E "node.*error|400" server.log
```

### 客户端日志关键字

```bash
# 查看爬取失败日志
grep -E "crawl.*fail|WAF|blocked" client.log

# 查看任务执行日志
grep "task_xxx" client.log
```
