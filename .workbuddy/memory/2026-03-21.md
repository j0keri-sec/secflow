## 文章图片 Base64 存储方案 (12:48)

### 需求
用户要求文章热点列表显示本地图片，而不是第三方链接。

### 方案对比
1. **原方案（文件上传）**：客户端下载 → 上传到服务端 → 返回本地 URL → 存储 URL
2. **新方案（Base64 直存）**：客户端下载 → 转 Base64 → 直接随文章 JSON 提交 → MongoDB string 字段存储

### 最终实现
修改 `secflow-client/pkg/articlegrabber/qianxin_weekly.go`：

```go
// downloadImageAsBase64 下载图片并转为 data URI
func (g *QianxinWeeklyGrabber) downloadImageAsBase64(imageURL string) string {
    // 1. HTTP 请求下载图片
    // 2. 读取图片二进制数据
    // 3. base64.StdEncoding.EncodeToString() 编码
    // 4. 返回 "data:image/png;base64,xxxx" 格式
}

// createArticleFromItem 中调用
if imageURL != "" && strings.HasPrefix(imageURL, "http") {
    imageBase64 = g.downloadImageAsBase64(imageURL)
}
```

### 验证结果
- ✅ 编译通过
- ✅ 图片下载成功 (530KB PNG → ~530k base64)
- ✅ 前端直接用 `<img :src="article.image">` 即可显示

### 数据库影响
- MongoDB string 类型支持 Base64（单个文档最大 16MB）
- Base64 编码后大小增加约 33%
- 平均每篇文章图片 ~700KB，编码后 ~930KB
- 20篇文章约 20MB，属于可接受范围
