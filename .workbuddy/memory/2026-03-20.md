## 奇安信安全周报爬虫修复 (22:16 - 23:16)

### 问题诊断
用户反馈点击列表项后会跳转到新页面，但之前爬虫无法获取详情内容。

### 根本原因
1. 奇安信网站的列表项没有 `<a>` 标签，无法直接提取链接
2. 点击列表项会在**新标签页**打开详情页
3. 原代码使用 JS `click()` 但没有处理新标签页

### 解决方案
1. 使用 `browser.Pages()` 获取所有打开的页面
2. 遍历找到 URL 包含 "notice-detail" 的新页面
3. 在新页面上提取内容
4. 关闭新页面，继续处理下一个

### 代码修改
1. **rodutil/browser.go**: 添加 `GetBrowserPages()` 函数获取所有页面
2. **articlegrabber/qianxin_weekly.go**: 重写 `Fetch()` 方法：
   - 点击列表项 → 打开新标签页
   - 遍历 `browser.Pages()` 找到详情页
   - 提取内容 → 关闭详情页 → 继续下一个

### 测试结果
```
成功提取 2 篇文章:
- 文章1: 11247 chars, URL: https://ti.qianxin.com/vulnerability/notice-detail/1763?type=hot-week
- 文章2: 10437 chars, URL: https://ti.qianxin.com/vulnerability/notice-detail/1755?type=hot-week
```

### 技术要点
- rod 原生 `Click()` 方法才能触发新标签页打开
- JS `click()` 不会打开新标签页（会触发当前页面的事件处理）
- 需要使用 `browser.Pages()` 访问新标签页

## 2026-03-20 23:30 - qianxin_weekly爬取器二次修复

### 新问题发现
1. 数据库检查发现奇安信热点周报文章数据有问题：
   - `url`字段为空字符串
   - `content`字段为空或包含错误内容（重复的列表页面内容）
   - `author`字段为空
   - 图片URL提取不完整

2. 从数据库中删除了2条错误的奇安信文章数据

### 根本原因分析
1. **新标签页处理复杂且不稳定**：之前的修复虽然能获取详情页，但处理流程复杂，容易出错
2. **页面跳转逻辑不一致**：网站可能在不同时间使用不同的跳转方式（当前页跳转 vs 新标签页）
3. **内容提取不可靠**：详情页的内容提取选择器可能随时间变化

### 新的修复方案（简化版）
1. **放弃详情页提取**：改为直接从列表页面提取所有可用信息
2. **列表页数据提取**：从列表项中提取标题、时间、摘要、图片和URL
3. **数据质量优先**：确保提取的数据准确可靠，即使不完整也比错误数据好

### 修复实现
1. 重写`Fetch`方法：
   - 专注于列表页面信息提取
   - 使用JavaScript从DOM中提取结构化数据
   - 添加多个选择器提高兼容性

2. 改进数据创建：
   - `createArticleFromItem`方法规范化数据创建
   - 验证和清理所有字段
   - 为缺失字段提供合理的默认值

3. 日期解析增强：
   - 支持多种日期格式
   - 自动添加当前年份处理不完整的日期
   - 更好的错误处理
